
# Aclaración de numeración de carpetas

Las carpetas del proyecto se encuentran numeradas con el fin de indicar la secuencia temporal de desarrollo de cada bloque de manera independiente.

El control de versiones detallado se gestiona mediante el sistema de versionado de GitHub, por lo que no se replican versiones dentro de la estructura de carpetas. En su lugar, **se mantiene un único registro de cambios en el log general del proyecto**, donde se documentan las modificaciones relevantes realizadas a lo largo del desarrollo.
# Lista BOM
En los siguientes listados de BOM por bloque se presentan los **componentes asociados a cada etapa** del sistema.

Dado que algunos elementos son compartidos entre múltiples bloques (por ejemplo, el microcontrolador), estos pueden aparecer en más de una lista. En dichos casos, la cantidad se indica con el **símbolo "~", el cual señala que el componente ya ha sido contabilizado** previamente en otra sección y no debe sumarse nuevamente al total del sistema.

Por lo tanto, las listas por bloque tienen un carácter descriptivo y funcional, mientras que el conteo total de componentes se realiza únicamente en la lista BOM global.
